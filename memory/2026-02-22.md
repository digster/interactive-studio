## Update - Python Run Target Resolution for Cross-Project Tabs

Fixed a production bug where running a Python file could fail with `No such file or directory` after switching projects while a tab from another project remained active.

### Root Cause

- Python execution used:
  - `activeProject.path` from the project selector
  - `activeTab.name` from the currently open tab
- When these came from different projects (for example, active tab `python-dash/app.py` while selected project `python-fastapi`), backend execution attempted a non-existent path like `python-fastapi/app.py`.

### Fix

- Updated `src/hooks/useCodeExecution.ts`:
  - Added execution target resolution from `activeTab.path`.
  - Detects the owning project by matching the tab path against known workspace project roots.
  - Passes a computed relative script path to `run_python` / `run_python_app` instead of using only `activeTab.name`.

### Tests

- Updated `tests/unit/hooks/useCodeExecution.test.ts`:
  - Added regression test: `runs using the tab project when selected project is different`.
  - Verifies app-mode execution uses the tab's project path and script, not the selected project path.

### Documentation and Meta

- Updated `ARCHITECTURE.md` to document execution target resolution behavior in `useCodeExecution`.
- Appended latest prompt to `PROMPT.md`.
