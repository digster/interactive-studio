# 2026-05-07

## Console + Problems for JavaScript / TS / JSX / TSX, plus Tab Size fix

### Why
- Console and Problems tabs were Python-only. JS code in the iframe preview ran silently ‚Äî no `console.log` capture, no error reporting.
- Tab Size 2/4 setting existed in the UI but had no real effect: CodeMirror's `EditorState.tabSize` only controls the *visual* width of `\t`, not what gets inserted on Tab/auto-indent.

### Changes
- **`src/lib/previewBridge.ts`** (new) ‚Äî `BRIDGE_SCRIPT` is an inline IIFE injected into every preview HTML. Wraps `console.log/info/warn/error/debug`, attaches `error` + `unhandledrejection` listeners, safely serializes args (Errors, BigInt, circular refs, functions), and posts `{ __previewBridge: true, kind, ... }` envelopes to `window.parent`. `handlePreviewMessage` validates the marker and routes to `addConsoleEntry` / `addProblem`. Uses `targetOrigin: '*'` because sandboxed iframes get an opaque `null` origin ‚Äî no stricter target works there.
- **`src/lib/previewTranspile.ts`** (new) ‚Äî Lazy-init `esbuild-wasm` (memoized via module-level promise; failed init clears the cache for retry). `transpile(code, language, absoluteFilePath)` runs `esbuild.transform` with `loader: 'ts' | 'tsx' | 'jsx'`, `sourcefile`, `sourcemap: 'inline'`, `format: 'esm'`, `jsx: 'automatic'`. Output gets `//# sourceURL=<abs>` appended. Errors normalize to `{ message, file, line, column }` with column bumped 0‚Üí1-based.
- **`src/lib/previewBuilder.ts`** ‚Äî `inlineAssets` now branches per inlined script's language: plain `.js` gets `//# sourceURL=`, TS/JSX/TSX gets transpiled and emitted as `<script type="module">`. Transpile failures become an inert `window.parent.postMessage(...)` so the error surfaces as a Problem instead of breaking the preview. `injectBridge()` inserts `BRIDGE_SCRIPT` immediately after `<head>` (or prepends if no head).
- **`src/hooks/useCodeExecution.ts`** ‚Äî added a `useEffect` that registers `handlePreviewMessage` on `window` (not Tauri-gated; works in browser dev too). Web-language Run path now also `clearConsole()`/`clearProblems()` and auto-opens the bottom panel to Console, mirroring Python.
- **`src/components/preview/PreviewPane.tsx`** ‚Äî `onLoad={() => { clearConsole(); clearProblems(); }}` on the html iframe so each reload starts clean. Scoped to html only; PythonPreview's iframe is unaffected.
- **`src/components/editor/CodeEditor.tsx`** ‚Äî Imported `indentUnit` from `@codemirror/language`. New `indentUnitCompartment` configured to `indentUnit.of(' '.repeat(tabSize))`. Reconfigure effect now dispatches both `tabSize` and `indentUnit` together. The Settings panel UI was unchanged ‚Äî the 2/4 segmented control was already wired to `settingsStore.tabSize`.
- **`package.json`** ‚Äî added `esbuild-wasm@0.28.0`.
- **Tests** ‚Äî `previewBridge.test.ts` (6) + `previewTranspile.test.ts` (3) + 3 new cases in `previewBuilder.test.ts` (bridge injection, sourceURL pragma, TSX transpile path, transpile-failure surface) + 1 new CodeEditor test (reconfigure preserves view identity). 199/199 pass (was 186).

### Notes for future me
- The bridge envelope marker `__previewBridge: true` is the *only* trust check on inbound `postMessage` data ‚Äî origin checks won't work because the sandbox forces an opaque origin. Don't add origin filters expecting a particular value, they'll always be `'null'`.
- Vitest jsdom tests cannot exercise real esbuild-wasm (no wasm runtime). All transpile tests mock `esbuild-wasm` directly; `previewBuilder` tests also mock `previewTranspile`. Keep that in mind when adding tests in this area.
- `esbuild-wasm` Vite import: `import wasmURL from 'esbuild-wasm/esbuild.wasm?url'`. Vite copies the wasm to the output and gives us a real URL. No vite.config.ts changes needed in this stack.
- For TS/JSX preview to *render* (not just transpile), the user's HTML must provide an import map for runtime modules (`react`, `react-dom`). esbuild `transform` does not bundle. This is documented as a follow-up in the plan; bare `import 'react'` will fail at runtime, surfacing as a Problem entry ‚Äî the bridge still works.
- A console-bridge in a tight loop (`for (;;) console.log(...)`) will flood the store. No cap was added; relying on `onLoad` clear at next reload is the current strategy.
