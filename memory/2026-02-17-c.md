# 2026-02-17 (c) - Python File Preview Support

## Summary

Added Python execution output rendering to the preview pane. Previously, Python files had `previewType: 'none'` so the preview showed an empty placeholder. Now Python stdout appears in the preview pane with auto-detection for HTML output.

## Changes

### 1. executionStore.ts
- Added `'python'` to `PreviewType` union
- Added `pythonOutput: string` (accumulated stdout) and `pythonOutputReady: boolean` state
- Added actions: `appendPythonOutput(data)`, `clearPythonOutput()`, `setPythonOutputReady(ready)`

### 2. languageDetect.ts
- Added `case 'python': return 'python'` in `getPreviewType()`
- Updated return type to include `'python'`

### 3. usePreview.ts
- When `pvType === 'python'`, sets `previewType` to `'python'` but skips `clearPreview()` to preserve last execution output

### 4. useCodeExecution.ts
- `python-output` listener (stdout): calls `appendPythonOutput(data)` alongside existing `addConsoleEntry`
- `python-exit` listener: calls `setPythonOutputReady(true)`
- `execute()`: calls `clearPythonOutput()` and `setPreview('', 'python')` before starting

### 5. PreviewPane.tsx
- Extracted `PythonPreview` component
- `looksLikeHtml()` helper: checks if output starts with `<!doctype` or `<html`
- States: running spinner, "Run to see preview" placeholder, HTML iframe, text pre with blinking cursor
- Theme-aware using CSS variables

### 6. Tests
- Updated `executionStore.test.ts` — 4 new tests for pythonOutput state/actions
- Updated `PreviewPane.test.tsx` — 7 new tests for Python rendering (HTML auto-detect, text, placeholders, running state)
- Created `languageDetect.test.ts` — 8 tests for getPreviewType, detectLanguage, isExecutable, getLanguageLabel

### 7. Architecture
- Updated `ARCHITECTURE.md` sections: executionStore, PreviewPane, Python Execution data flow
