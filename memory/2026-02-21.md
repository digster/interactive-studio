# 2026-02-21 - Dash App Mode and Example

## Summary

Implemented full Dash support path for Interactive Studio with embedded preview and process lifecycle controls.

## Backend (Tauri / Rust)

- Added managed Python app lifecycle in `src-tauri/src/commands/python.rs`:
  - New commands: `run_python_app`, `stop_python_app`
  - New event: `python-app-ready` with app URL payload
  - Added `PythonProcessManager` managed state to track one active long-running Python app process
  - Added readiness probing on localhost and port auto-selection (starting from 8050, fallback range)
  - Added stdout/stderr streaming for long-running apps and proper `python-exit` emission
- Kept existing one-shot `run_python` behavior for normal scripts
- Wired commands/state in `src-tauri/src/lib.rs`

## Frontend (React / Zustand)

- Extended execution state in `src/store/executionStore.ts`:
  - Added `previewUrl`
  - Added `runningMode` (`script` | `app` | `null`)
  - Added actions `setPreviewUrl` and `setRunningMode`
- Added Tauri wrappers in `src/lib/tauriFS.ts`:
  - `runPythonApp(...)`
  - `stopPythonApp()`
- Updated `src/hooks/useCodeExecution.ts`:
  - Detects Dash-like files and runs app mode via `run_python_app`
  - Handles `python-app-ready` event and switches preview to URL iframe mode
  - Adds stop behavior via `stop_python_app`
  - Preserves script-mode Python output behavior
- Updated `src/hooks/usePreview.ts` to avoid overriding URL preview while app mode is active
- Updated `src/components/preview/PreviewPane.tsx` with URL preview iframe mode and waiting state
- Updated `src/components/editor/EditorPane.tsx` with Stop button + shortcut behavior for running app mode
